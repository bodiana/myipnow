DNS Lookup

Instantâneo DNS Lookup para verificar os registos A, AAAA, MX, CNAME e TXT de qualquer domínio. Encontre endereços IP e detalhes de DNS rapidamente.

O que é um DNS Lookup?

Um DNS Lookup é uma consulta que pergunta ao Sistema de Nomes de Domínio quais os registos que um domínio publica - como A/AAAA (endereços IP), MX (servidores de correio eletrónico), NS (servidores de nomes autoritativos) e TXT (verificação/SPF/DKIM). É utilizado para solucionar problemas de encaminhamento, entrega de correio eletrónico e verificação da propriedade do domínio.

Como utilizar este DNS Lookup

  1. Introduza um domínio (example.com).
  2. Selecione o tipo A record (A, AAAA, MX, NS, TXT).
  3. Clique em Lookup para ver os resultados DNS em direto.

Explicação dos tipos DNS record

  • A: Mapeia um nome de domínio para um endereço IPv4.
  • AAAA: Mapeia um nome de domínio para um endereço IPv6.
  • MX: Registo Mail Exchange, utilizado para servidores de correio eletrónico.
  • NS: Registo de servidor de nomes, identifica servidores DNS autorizados.
  • TXT: Registo de texto, frequentemente utilizado para verificação ou definições SPF/DKIM.

Perguntas mais frequentes

O que é um DNS Lookup e porque é que é importante?
Um DNS Lookup ajuda a identificar a forma como os nomes de domínio são resolvidos para endereços IP, garantindo que o seu sítio Web e o seu correio eletrónico chegam aos servidores corretos.
Como é que verifico os DNS records do meu domínio?
Introduza o seu domínio e selecione o tipo DNS record para ver as informações A, AAAA, MX, NS e TXT em tempo real.
O que significam os registos A, MX, NS e TXT?
A records mapeiam o domínio para IPv4, AAAA para IPv6, MX para servidores de correio, NS para servidores de nomes, TXT para verificação e registos SPF/DKIM.
Porque é que o meu domínio não apresenta registos?
Alguns domínios podem ter proteção de privacidade ou podem ainda não ter determinados DNS records configurados.
O que é a propagação DNS?
DNS propagação é o tempo que leva para que as alterações de DNS record se espalhem por todos os servidores do mundo. Isso pode levar de alguns minutos a 48 horas, dependendo das configurações de TTL.
Como posso solucionar problemas de entrega de correio eletrónico com DNS?
Verifique os MX records para verificar se eles apontam para o servidor de email correto. Verifique também os registos TXT para entradas SPF e DKIM. Registos em falta ou incorrectos são uma causa comum de falhas na entrega de correio eletrónico.
O que é um CNAME record?
Um registo CNAME (Canonical Name) mapeia um nome de domínio para outro. É normalmente utilizado para apontar subdomínios como www para o domínio raiz, ou para apontar para serviços externos como CDNS.

O que é DNS e como funciona?

O DNS (Domain Name System) é frequentemente designado por lista telefónica da Internet. Ele traduz nomes de domínio legíveis por humanos, como example.com, em endereços IP que os computadores usam para se comunicar. Sem o DNS, seria necessário memorizar o endereço IP de cada sítio Web que se visita.

Quando escreve um domínio no seu browser, o seu dispositivo consulta um resolvedor DNS, que procura a resposta nos servidores de nomes autorizados e devolve o endereço IP. Todo este processo demora, normalmente, milissegundos.

Que tipos de DNS record posso verificar?

Os tipos mais comuns de DNS record são: A records (mapear o domínio para IPv4), AAAA records (mapear para IPv6), MX records (especificar servidores de correio), NS records (identificar servidores de nomes autoritativos), Registos TXT (utilizados para SPF, DKIM e verificação de domínios) e CNAME records (alias de um domínio para outro).

Como utilizar DNS Lookup para resolver problemas de correio eletrónico

Se os emails não estiverem sendo entregues, verifique os MX records do seu domínio para verificar se eles apontam para o servidor de email correto. Verifique também os registos TXT para as entradas SPF e DKIM que impedem que os emails sejam marcados como spam. Os registos em falta ou mal configurados são a causa mais comum de falhas na entrega de correio eletrónico.

O que é a propagação DNS?

DNS propagação é o tempo que leva para que as alterações de DNS record se espalhem por todos os servidores DNS do mundo. Isso pode levar de alguns minutos a 48 horas. O valor TTL (Time to Live) em cada registo controla durante quanto tempo é colocado em cache. Se você alterou recentemente os DNS records e eles não estão sendo exibidos corretamente, a propagação pode ainda estar em andamento.
