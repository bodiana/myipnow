DNS Lookup Tool

Instant DNS Lookup zur Überprüfung von A-, AAAA-, MX-, CNAME- und TXT-Einträgen für beliebige Domains. Finden Sie schnell IP-Adressen und DNS-Details.

Was ist ein DNS Lookup?

Ein DNS Lookup ist eine Abfrage, die das Domänennamensystem fragt, welche Einträge eine Domäne veröffentlicht - wie A/AAAA (IP-Adressen), MX (Mailserver), NS (autoritative Nameserver) und TXT (Verifizierung/SPF/DKIM). Es wird für die Fehlersuche beim Routing verwendet, E-Mail-Zustellung und Überprüfung des Domänenbesitzes.

Wie wird dieser DNS Lookup verwendet?

  1. Geben Sie eine Domäne ein (example.com).
  2. Wählen Sie den Typ A record (A, AAAA, MX, NS, TXT).
  3. Klicken Sie auf Suchen, um die Live-Ergebnisse von DNS zu sehen.

Verwenden Sie dieses DNS Lookup-Tool, um sofort die DNS records einer beliebigen Domain zu überprüfen. Dies hilft bei der Identifizierung von IP-Adressen, Mailservern, autoritativen Nameservern und Verifizierungseinträgen für Ihre Domain.

DNS record-Typen erklärt

  • A: Ordnet einen Domänennamen einer IPv4-Adresse zu.
  • AAAA: Ordnet einen Domänennamen einer IPv6-Adresse zu.
  • MX: Mail Exchange-Eintrag, verwendet für E-Mail-Server.
  • NS: Name Server record, identifiziert maßgebliche DNS-Server.
  • TXT: Textdatensatz, der häufig zur Überprüfung oder für SPF/DKIM-Einstellungen verwendet wird.

Häufig gestellte Fragen

Kann ich DNS Lookup für jede Domain verwenden?
Ja. Geben Sie einen Domainnamen ein, um die DNS-Einträge abzurufen. Sie können Ihre eigene Domain, Konkurrenz-Domains oder jede öffentliche Domain prüfen.
Was passiert, wenn ein DNS-Eintrag gelöscht wird?
Wenn ein DNS-Eintrag gelöscht wird, liefern Resolver, die ihn gecacht haben, weiterhin den alten Wert bis der TTL abläuft. Danach geben sie kein Ergebnis für diesen Eintragstyp zurück. E-Mail-Zustellung und Website-Zugang können betroffen sein.
Was bedeuten A-, MX-, NS- und TXT-Einträge?
A-Einträge verweisen eine Domain auf eine IP-Adresse. MX-Einträge definieren Mailserver. NS-Einträge geben autoritative Nameserver an. TXT-Einträge speichern Text für SPF, DKIM und Domain-Verifizierung.
Warum zeigt meine Domain keine Einträge?
DNS-Änderungen können bis zu 48 Stunden dauern, um sich global zu verbreiten. Wenn Ihre Domain keine Einträge zeigt, könnte sie neu registriert, kürzlich transferiert oder mit einem falsch konfigurierten Nameserver eingerichtet sein.
Warum sehen verschiedene Standorte unterschiedliche DNS-Ergebnisse?
DNS-Einträge werden von Resolvern weltweit mit unterschiedlichen TTL-Ablaufzeiten gecacht. Nach einer DNS-Änderung sehen manche Standorte sofort die neuen Einträge, während andere noch die alten sehen. Dies ist normal und löst sich innerhalb von 24-48 Stunden.
Was ist der Unterschied zwischen autoritativem und rekursivem DNS?
Ein autoritativer DNS-Server enthält die tatsächlichen DNS-Einträge einer Domain. Ein rekursiver DNS-Server sucht Einträge für Sie, indem er autoritative Server abfragt. Ihr ISP und öffentliche Resolver wie Google (8.8.8.8) verwenden rekursives DNS.
Was ist ein CNAME-Eintrag?
Ein CNAME-Eintrag (Canonical Name) ordnet einen Domainnamen einem anderen zu. Er wird häufig für Subdomains wie www oder mail verwendet, um auf eine primäre Domain oder einen CDN-Hostnamen zu verweisen.

Was ist DNS und wie funktioniert es?

DNS (Domain Name System) wird oft als das Telefonbuch des Internets bezeichnet. Es übersetzt menschenlesbare Domänennamen wie example.com in IP-Adressen, die Computer zur Kommunikation verwenden. Ohne DNS müssten Sie sich die IP-Adresse jeder Website, die Sie besuchen, merken.

Wenn Sie eine Domäne in Ihren Browser eingeben, fragt Ihr Gerät einen DNS-Resolver ab, der die Antwort von maßgeblichen Namensservern abruft und die IP-Adresse zurückgibt. Dieser gesamte Vorgang dauert in der Regel einige Millisekunden.

Welche DNS record-Typen kann ich überprüfen?

Die häufigsten DNS record-Typen sind: A records (bilden die Domain auf IPv4 ab), AAAA records (bilden auf IPv6 ab), MX records (geben Mailserver an), NS records (identifizieren autoritative Nameserver), TXT-Records (werden für SPF, DKIM und die Überprüfung von Domänen verwendet) und CNAME records (Alias einer Domäne für eine andere).

So verwenden Sie DNS Lookup zur Fehlerbehebung bei E-Mails

Wenn E-Mails nicht zugestellt werden, überprüfen Sie die MX record-Einträge für Ihre Domäne, um sicherzustellen, dass sie auf den richtigen Mailserver verweisen. Überprüfen Sie auch die TXT-Einträge auf SPF- und DKIM-Einträge, die verhindern, dass E-Mails als Spam markiert werden. Fehlende oder falsch konfigurierte Einträge sind die häufigste Ursache von E-Mail-Zustellungsfehlern.

Was bedeutet DNS-Vermehrung?

Die DNS-Ausbreitung ist die Zeit, die benötigt wird, um DNS record-Änderungen auf alle DNS-Server weltweit zu übertragen. Dies kann zwischen einigen Minuten und 48 Stunden dauern. Der TTL-Wert (Time to Live) für jeden Datensatz bestimmt, wie lange er im Cache gespeichert wird. Wenn Sie kürzlich DNS records geändert haben und diese nicht korrekt angezeigt werden, ist die Übertragung möglicherweise noch im Gange.
