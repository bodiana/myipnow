DNS Lookup

DNS Lookup istantaneo per controllare i record A, AAAA, MX, CNAME e TXT di qualsiasi dominio. Trova rapidamente indirizzi IP e dettagli DNS.

Che cos'è un DNS Lookup?

Una DNS Lookup è un'interrogazione che chiede al Domain Name System quali record pubblica un dominio, come A/AAAA (indirizzi IP), MX (server di posta), NS (server di nomi autorevoli) e TXT (verifica/SPF/DKIM). Viene utilizzato per risolvere i problemi di instradamento, consegna delle e-mail e verifica della proprietà del dominio.

Come utilizzare questo DNS Lookup

  1. Immettere un dominio (example.com).
  2. Selezionare il tipo di A record (A, AAAA, MX, NS, TXT).
  3. Fare clic su Cerca per vedere i risultati in tempo reale di DNS.

Spiegazione dei tipi di DNS record

  • A: Mappa un nome di dominio a un indirizzo IPv4.
  • AA: Mappa un nome di dominio a un indirizzo IPv6.
  • MX: record di Mail Exchange, utilizzato per i server di posta elettronica.
  • NS: record Name Server, identifica i server DNS autorevoli.
  • TXT: Record di testo, spesso usato per la verifica o per le impostazioni di SPF/DKIM.

Domande frequenti

Posso usare DNS Lookup per verificare qualsiasi dominio?
Sì. Inserisci qualsiasi nome di dominio per recuperare i suoi record DNS. Puoi controllare il tuo dominio, i domini dei concorrenti o qualsiasi dominio pubblico.
Cosa succede quando un record DNS viene eliminato?
Quando un record DNS viene eliminato, i resolver che lo hanno memorizzato nella cache continueranno a servire il vecchio valore fino alla scadenza del TTL. Dopodiché non restituiranno alcun risultato per quel tipo di record. La consegna delle email e l'accesso al sito possono essere interessati.
Cosa significano i record A, MX, NS e TXT?
I record A puntano un dominio a un indirizzo IP. I record MX definiscono i server di posta. I record NS specificano i nameserver autoritativi. I record TXT memorizzano testo per SPF, DKIM e la verifica del dominio.
Perché il mio dominio non mostra record?
Le modifiche DNS possono richiedere fino a 48 ore per propagarsi globalmente. Se il tuo dominio non mostra record, potrebbe essere appena registrato, trasferito di recente o avere un nameserver mal configurato.
Perché posizioni diverse vedono risultati DNS diversi?
I record DNS vengono memorizzati nella cache dai resolver di tutto il mondo con diversi tempi di scadenza TTL. Dopo una modifica DNS, alcune posizioni vedono immediatamente i nuovi record mentre altre vedono ancora quelli vecchi. È normale e si risolve entro 24-48 ore.
Qual è la differenza tra DNS autoritativo e ricorsivo?
Un server DNS autoritativo contiene i record DNS effettivi di un dominio. Un server DNS ricorsivo cerca i record per conto tuo interrogando i server autoritativi. Il tuo ISP e i resolver pubblici come Google (8.8.8.8) usano il DNS ricorsivo.
Cos'è un record CNAME?
Un record CNAME (Nome Canonico) mappa un nome di dominio su un altro. È comunemente usato per sottodomini come www o mail per puntare a un dominio principale o a un hostname CDN.

Che cos'è DNS e come funziona?

DNS (Domain Name System) è spesso chiamato l'elenco telefonico di Internet. Traduce nomi di dominio leggibili dall'uomo come example.com in indirizzi IP che i computer utilizzano per comunicare. Senza DNS, sarebbe necessario memorizzare l'indirizzo IP di ogni sito web visitato.

Quando si digita un dominio nel browser, il dispositivo interroga un resolver DNS, che cerca la risposta dai server dei nomi autorevoli e restituisce l'indirizzo IP. L'intero processo dura in genere millisecondi.

Quali tipi di DNS record è possibile controllare?

I tipi di DNS record più comuni sono: A records (mappa il dominio a IPv4), AAAA records (mappa a IPv6), MX records (specifica i server di posta), NS records (identificano i server dei nomi autorevoli), registri TXT (usati per SPF, DKIM e per la verifica dei domini) e CNAME records (alias di un dominio a un altro).

Come utilizzare DNS Lookup per la risoluzione dei problemi di posta elettronica

Se le e-mail non vengono consegnate, controllate gli MX record del vostro dominio per verificare che puntino al server di posta corretto. Controllate anche i record TXT per le voci SPF e DKIM che impediscono alle e-mail di essere contrassegnate come spam. I record mancanti o mal configurati sono la causa più comune di errori di consegna delle e-mail.

Che cos'è la propagazione DNS?

La propagazione di DNS è il tempo necessario affinché le modifiche di DNS record si diffondano su tutti i server DNS nel mondo. Può richiedere da pochi minuti a 48 ore. Il valore TTL (Time to Live) di ciascun record controlla il tempo di permanenza nella cache. Se si sono modificati di recente i DNS record e questi non vengono visualizzati correttamente, è possibile che la propagazione sia ancora in corso.
